**English translation:**

1. Provide answers in a simple format—no essays about the problem, just straight to the point.
2. Give help in the form of complete function code or blocks, and clearly state which function is to be replaced—provide the name of the function to be swapped.
3. If a conditional block is to be replaced, specify where it is located—do it simply, referring to the function name and the place inside it.
4. Respect the existing comments in the code.
5. If new functions are introduced, always include a functional description in a code comment, 1–2 lines max, e.g. `/* ... */`.
6. Never use comments to describe code edits, e.g. `/* here */ /* new code */` — comments must serve as anchors in the code and mark functional blocks that will later be easier to modify or replace.
7. Do not create fragmented code in the form of small helper functions if it is better to just expand the given function.
8. Do not remove code that is not understood or make improvements that are not fully understood if the change could alter how something else works.
